AI Generation

UI Integration Cards are declarative, which makes them well-suited for generation by Large Language Models (LLMs). With the right tools and context, AI coding agents can scaffold new cards, fix issues in existing ones, and follow current best practices end-to-end.

UI5 provides two ways to enable AI-driven card development:

  • UI5 Plugins for Coding Agents (Recommended): bundles MCP tools and skills for Claude Code.
  • UI5 MCP Server: for any agent that supports MCP.

UI5 Plugins for Coding Agents (Recommended)

The UI5 Plugins for Coding Agents are the recommended setup for AI-driven UI Integration Cards development. They bundle the UI5 MCP Server tools together with the Integration Cards skill described below, guiding the agent through the full lifecycle, from scaffolding a new card to validating its manifest and previewing it locally.

Installation

See the plugin README for installation instructions.

Integration Cards Skill

The plugin includes the ui5-best-practices-integration-cards skill, which is loaded automatically by the agent when working on UI Integration Cards and enforces the patterns required for production-quality cards.

UI5 MCP Server

The UI5 MCP Server is the engine behind the UI5 Plugins for Coding Agents and can also be used standalone with any agent that supports the Model Context Protocol. It exposes the following tools:

  • get_integration_cards_guidelines: provides dedicated guidelines for the development of UI Integration Cards.
  • create_integration_card: creates new UI Integration Cards based on specified parameters.
  • run_manifest_validation: validates the manifest.json file for UI Integration Cards, as well as regular UI5 applications and libraries.

Standalone Setup

To use the MCP server directly, without the UI5 Plugins for Coding Agents, follow the UI5 MCP Server setup instructions.

Benefits

Using the UI5 plugins or the MCP server to develop UI Integration Cards offers several key advantages:

  • Up-to-date Best Practices: LLMs gain access to the latest recommended APIs and development patterns for UI Integration Cards, ensuring your cards follow current standards.
  • Version Compatibility: Prevents the use of deprecated properties or APIs that are not available in your project's version, reducing errors and technical debt.
  • Efficient Scaffolding: Helps AI agents start new projects faster with established patterns, saving development time and token costs.
  • Card-Type Specific Knowledge: Provides specialized knowledge for each card type. For example, it can suggest the best-fitting chart types for an Analytical card.
  • Cross-Platform Compatibility: Encourages the use of Declarative card types to ensure compatibility with both web and mobile native platforms.

How it Works

The tools above build on a small set of openly published resources. If you are building your own AI integration (a custom agent, an editor extension, or a different MCP server), you can reuse the same building blocks directly:

  • Development guidelines: the UI Integration Cards Development Guidelines provide the context an LLM needs to follow current best practices. Drop them into your own system prompt or skill.
  • Card template: the card template gives the LLM a vetted starting point so it does not have to invent the manifest skeleton from scratch.
  • Manifest schema: the UI5 Manifest schema lets your tooling validate generated manifests and surface concrete errors back to the model.
